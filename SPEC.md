# Todo 日程提醒机器人 - 项目规格说明书

## 1. 项目概述

### 项目名称
**小管家 Todo** - 智能日程管理助手

### 核心定位
一个温暖的每日安排预览系统，支持多视图展示（单日/周/月），可与飞书群机器人实时互动，完成任务后记录感受。

### 目标用户
- 需要管理日常琐事、学习工作安排的职场人士和学生
- 习惯在飞书群中协作办公的用户
- 希望以温暖、清新风格管理日程的用户

## 3. 数据库设计

### 数据表结构

#### 3.1 `todos` - 日程任务表

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | UUID | 主键 |
| title | TEXT | 任务标题 |
| description | TEXT | 任务描述（可选） |
| start_time | TIMESTAMPTZ | 开始时间（精确到分钟） |
| end_time | TIMESTAMPTZ | 结束时间（精确到分钟） |
| category | TEXT | 分类：life/daily/work/study |
| is_completed | BOOLEAN | 是否完成 |
| completed_at | TIMESTAMPTZ | 完成时间（可选） |
| feeling | TEXT | 完成后感受（可选） |
| created_at | TIMESTAMPTZ | 创建时间 |
| updated_at | TIMESTAMPTZ | 更新时间 |

#### 3.2 `bot_interactions` - 机器人交互记录表

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | UUID | 主键 |
| todo_id | UUID | 关联的任务ID |
| message_id | TEXT | 飞书消息ID |
| action | TEXT | 操作类型：complete/feelings/cancel |
| content | TEXT | 交互内容 |
| created_at | TIMESTAMPTZ | 交互时间 |

## 4. 功能模块

### 4.1 Web 端功能

#### 4.1.1 日程视图切换
- **单日视图**：展示当天所有日程，按时间线排列
- **周视图**：以周为单位的日历网格，显示每日任务数量
- **月视图**：月历形式，快速跳转到指定日期

#### 4.1.2 新建日程
- 标题（必填）
- 描述（可选）
- 开始时间（精确到分钟，日期时间选择器）
- 结束时间（精确到分钟）
- 分类选择：
  - 🏠 生活琐事（life）
  - ☀️ 日常提醒（daily）- 吃饭、睡觉、保健品等
  - 💼 工作事件（work）
  - 📚 学习任务（study）

#### 4.1.3 完成与感受记录
- 点击任务完成按钮
- 弹出感受选择：
  - 😊 很棒
  - 😐 一般
  - 😔 有点沮丧
  - 💪 充满能量
  - 🎉 超额完成
- 可输入自定义感受文字

#### 4.1.4 任务编辑与删除
- 编辑现有任务信息
- 删除任务（确认对话框）

### 4.2 飞书群机器人功能

#### 4.2.1 每日定时提醒
- 发送时间：每天早上 8:30
- 消息内容：
  ```
  🌅 早安！今日日程提醒

  📅 今日共 {n} 个事项

  ⏰ 时间线：
  1. 09:00-10:00 | 早会 | 💼 工作
  2. 12:00-13:00 | 午餐 | ☀️ 日常
  ...

  回复格式：
  - "完成 1" - 标记第1项完成
  - "完成 1 感觉很棒" - 标记完成并记录感受
  - "取消 1" - 取消标记
  ```

#### 4.2.2 交互式回复处理
- 解析用户回复
- 更新 Supabase 数据
- 发送确认消息

## 6. API 设计

### 6.1 Supabase Edge Functions

#### `GET /functions/v1/daily-reminder`
- 获取当日所有未完成任务
- 返回格式化的时间线数据

#### `POST /functions/v1/bot-webhook`
- 接收飞书机器人回调
- 解析用户消息
- 更新任务状态

### 6.2 客户端 API

| 端点 | 方法 | 说明 |
|------|------|------|
| /todos | GET | 获取日程列表 |
| /todos | POST | 创建新日程 |
| /todos/:id | PATCH | 更新日程 |
| /todos/:id | DELETE | 删除日程 |
| /todos/:id/complete | POST | 完成任务并记录感受 |

## 7. GitHub Actions 工作流

### 7.1 定时任务配置

```yaml
name: Daily Reminder Bot
on:
  schedule:
    - cron: '30 8 * * *'  # 每天 8:30 UTC
  workflow_dispatch:  # 支持手动触发
```

### 7.2 工作流步骤

1. 获取 Supabase 凭证
2. 调用 Edge Function 获取当日任务
3. 调用飞书 API 发送消息
4. 记录发送日志

## 8. 飞书机器人配置

### 8.1 消息卡片模板

使用飞书开放平台的卡片消息组件：
- 标题区：日期 + 问候语
- 时间线区：任务列表
- 操作提示区：使用说明

### 8.2 Webhook 配置

- 事件订阅：接收消息回调
- 消息解密：使用 AES 加密
- 回调验证：签名校验
